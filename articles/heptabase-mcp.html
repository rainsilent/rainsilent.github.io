讓 Heptabase 變成 AI 也能使用的第二大腦：Heptabase MCP

之前我寫過，Heptabase 對我來說很像收藏腦袋記憶的地方。

很多想法一開始都不是完整文章，而是片段、筆記、會議紀錄、技術資料、突然想到的靈感。

透過 Heptabase 的卡片、Journal、Tag 和白板，可以先把腦袋裡的東西卸載出來，再慢慢整理成有關聯的知識。

不過實際工作時，資訊來源不會只在筆記本裡。

Email、網頁、PDF、內部文件、知識庫、聊天紀錄，每個地方都有一部分資訊。

我一直希望 AI 也可以從我的記事本萃取我的筆記內容，當成知識庫來協助我工作。

那要怎麼讓 Heptabase 跟 AI 互動？

目前 Heptabase 已經提供 MCP。

簡單來說，MCP 有點像是 AI 時代的標準串接規格，概念上類似傳統 API：大家先講好溝通方式，AI 工具就可以透過這個規格和 Heptabase 互動。

目前 Heptabase MCP 支援的功能包含：

  • 搜尋 Heptabase 裡的 note cards、journals、PDF、web cards 等內容
  • 搜尋 whiteboards
  • 讀取 whiteboard 裡的卡片、文字、section、mindmap 等內容
  • 讀取指定的 note、journal、PDF、media transcript、highlight 等物件
  • 讀取一段日期範圍內的 journals
  • 搜尋 PDF 內容
  • 讀取 PDF 指定頁數
  • 將內容存成新的 note card
  • 將內容 append 到指定日期的 journal
  • 查詢 tags 與 tag 底下的 cards

啟用方式也很簡單，基本上就是在你使用的 AI 工具裡新增 Heptabase MCP，再完成授權。

如果是 ChatGPT / Codex，可以到 Apps & Connectors 的 MCP 設定新增 Heptabase MCP server，填入：

https://api.heptabase.com/mcp

然後選 OAuth 授權，登入 Heptabase 後允許存取即可。

如果是 Claude Code，也可以直接新增 MCP：

claude mcp add --transport http heptabase-mcp https://api.heptabase.com/mcp

接著在 Claude Code 裡做 MCP 授權。

設定完成後，AI 就可以在你允許的範圍內讀取或寫入 Heptabase。

這解決了一個我實際使用 AI 時很常遇到的問題：跟 AI 討論到一半，常常會產生一些值得記下來的資訊。

以前這種情境通常只能自己複製貼上，手動丟回筆記本。

但有了 MCP 之後，可以直接叫 AI：

「把這段整理成進筆記本。」

「把今天討論的結論加到記事本日記」

「幫我搜尋以前寫過的相關筆記，再整理成摘要。」

這樣對話就不只是停留在聊天紀錄裡，而是可以回到自己的知識系統中，成為下次可以繼續使用的資料。

對我來說，這在 Agent 自動化情境中特別有用。

例如 Agent 自動整理信件、追蹤任務、分析資料或產出每日摘要時，結果不一定要丟到通訊軟體。通訊軟體雖然即時，但資訊太雜，很容易被其他訊息淹沒。

Agent 可以透過 MCP 把重點直接寫回 Heptabase，早上我只要看 todo 和日記留下來的紀錄，就可以知道今天要處理什麼、哪些事情有風險、哪些資訊需要決策。

更進一步，也可以透過排程讓 AI 定期閱讀或整理 Heptabase 裡的內容，把重要資訊同步進 Agent 的知識庫。這樣 Agent 在工作時，就不只是根據當下的 prompt 回答，而是能帶著我平常累積的筆記、判斷邏輯和工作背景一起協助我。

例如定期整理最近的 Journal、抽取專案進度、更新常用知識、歸納待辦與風險。等到真正需要處理事情時，Agent 就能更快知道背景，也比較不需要每次重新解釋一次。

所以 Heptabase MCP 的價值不是「多一個 AI 功能」而已。

它真正重要的是，讓 Heptabase 從單純的個人筆記系統，變成 AI Agent 也能互動的知識工作區。以前是我把腦袋裡的東西存進 Heptabase；現在是 AI 也可以幫我搜尋、整理、記錄，讓這些資訊可以被持續使用。
