剛入門軟體業的 PM 可具備的基礎心法

計算機（電腦）基礎原理

計算機只看的懂0和1，所有資料，包括文字、圖片、影片等，都是以0和1的形式儲存的，甚至於跟電腦溝通的語言，轉換到最底層時，電腦的理解也都是靠0、1，而這就所謂的二進位。

程式碼

人類與電腦溝通的語言，告訴電腦要如何執行特定的任務， 常見的程式語言包括Java, C#, C++, Python等，基本上PM是不會需要自己動手寫的，寫程式的事情是由工程師完成，但需要了解功能需求的流程跟邏輯，才能有辦法跟工程師溝通。

Compile

將程式碼透過編譯器，轉換為電腦可以理解的機器碼（二進位檔案）的過程，像是C, C++這類編譯式語言，就需要經過compile才可以執行。而像是Python, javascript這種直譯式語言，就可以直接被使用（像是瀏覽器可以直接執行）。

Build

將編譯好的機器碼與其他相關的檔案（例如資源檔案、庫檔案等）打包成可執行檔案的過程。Build通常由建置工具（build tool）來完成，可能聽到的口語是「Build image」，也就是build出一個映像檔(image)，可以拿這個image去安裝程式。

應用程式（Application, App）

用於執行特定任務的軟體，例如文書處理、影音播放、遊戲等。 應用程式可以分為本地應用程式（單機版）和Web應用程式（網路版）。 本地應用程式需要安裝在電腦上才能使用，而Web應用程式可以直接在瀏覽器中使用。

資料庫（DB）

用於儲存和管理資料的軟體。資料庫可以分為結構化資料庫和非結構化資料庫。 結構化資料庫中的資料被組織成表格的形式，而非結構化資料庫中的資料則沒有固定的格式。

資料庫具象化就是一個excel，來儲存資料，只是他比excel大很多、複雜很多。

前端（Frontend）- 後端（Backend）

前端是指應用程式使用者可以直接看到的和操作的部分，例如網頁、介面等。 前端工程師們撰寫的會是使用者可以看見的內容，與UI設計師合作，將設計出的畫面實踐，也會撰寫javascript來實踐網頁上的動畫。 如涉及資料的部分，則會與後端工程師串接資料。

後端是指應用程式中負責處理資料和邏輯的部分，例如伺服器、資料庫等。 後端工程師除了撰寫上述內容，亦會與前端工程師合作，將資料透過如API的方式，提供給前端讀取。

Server — Client

Server（伺服器）和Client（客戶端）是兩種不同的電腦角色。

伺服器提供服務，而客戶端則使用這些服務。

在Web應用程式中，伺服器通常用於儲存資料和處理邏輯，而客戶端則用於顯示網頁和接收使用者輸入。

具體來說，會在實體的伺服器上架設Web service（像IIS, Apache），來提供應用程式（Server） 使用者就可以透過瀏覽器使用這個web應用程式，而這裡就會稱呼為Client。

API, CLI, etc.

結尾的”I”，都是指”Interface”。 那什麼是Interface呢？ 舉例：當前端畫面上需要特定資料（比方說使用者的生日時），就需要提供使用者帳號給後端，後端再依照使用者帳號查到的使用者生日，回傳給前端，讓前端可以呈現在畫面上。 而前、後端必須先講好，前端用什麼格式跟後端講，後端把資料回傳時要長什麼樣子，這就是interface的概念。

這樣講好規則後，前後端就不用每次要呼叫都要改。

現在最常見的就是API（Application Programming Interface），其實就是一種上面舉例的interface，允許不同的應用程式之間進行溝通，如網頁應用程式與後端的溝通。

CLI（Command-Line Interface），這是直接在電腦上的終端機（Terminal）下指令跟電腦溝通的方式（因為以前沒有圖形化介面）。如果要對電腦（機器）做一些進階操作，也會直接使用CLI做操作。
